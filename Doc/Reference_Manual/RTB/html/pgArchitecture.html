RTB Software Stack Architecture

The Ranging Toolbox is an addition to Atmel's IEEE 802.15.4 MAC package. For detailed information about the MAC layer see AVR2025 MAC Software Package.

Ranging Toolbox Architecture

The modularity of Atmel's MAC software implementation generally allows the use of different hardware. The hardware for the microcontroller and the board is abstracted by the Platform Abstraction Layer (PAL). The transceiver hardware is addressed by the Transceiver Abstraction Layer (TAL).

The Ranging Toolbox (RTB) is currently only implemented on a single handware platform - the ATxmega256A3 in conjunction with the AT86RF233 on REB233SMAD V8.1.0.

In the regular MAC stack architecture the MAC layer resides directly on top of the TAL layer. In order to meet the strict timing requirements for ranging measurements, the RTB is placed inbetween the MAC and the TAL layer as shown above.

The architecture allows for using both the MAC and the RTB in (quasi-)parallel, i.e. the application can either initiate a request to the MAC layer (i.e. MLME-/MCPS-Request/Response), or a request to the RTB (RTB-Request) in order to start a ranging procedure. The RTB simply enhances the existing MAC-API with further ranging related API functions (i.e. the RTB-API). As soon as the RTB is enabled in the software build, the same access mechanisms are provided by the RTB as known for the MAC layer.

The API layer automatically decides whether a request is routed to the regualar MAC or the RTB. In case a ranging procedure shall be intiated, the MAC temporarily does not process MAC related requests as long as the ranging procedure is ongoing.

Incoming frames are automatically forwarded from the TAL to the RTB, which in return decides whether the incoming frame is destined for the RTB or shall be forwarded to the MAC layer.
